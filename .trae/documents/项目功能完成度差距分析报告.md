根据对项目代码库的深度分析与项目文档的对比，以下是要求实现但尚未完成（或仅有占位符/Mock实现）的功能列表：

### 1. 核心交互与UI
- **节点生成进度遮罩**：
  - **文档要求**：节点生成时，节点卡片上必须显示**白色半透明遮罩**展示生成进度（stream token/percentage）。
  - **现状**：目前生成进度是在创建弹窗（`NodeCreateDialog`）中显示，生成完成后才关闭弹窗，未实现节点卡片上的实时进度遮罩。
- **思维导图功能**：
  - **文档要求**：内容界面左上角按钮调用LLM生成思维导图，再次点击唤出1/4大小的浮动面板进行编辑。
  - **现状**：代码中仅有按钮占位，点击提示“功能正在开发中”，相关数据结构和渲染组件均未实现。

### 2. AI 与 LLM 能力
- **AI 对话抽屉（Sidebar）**：
  - **文档要求**：对接真实LLM，Prompt必须包含**当前界面可见内容** + **相关节点树片段** + **联网检索摘要**；支持代码高亮、消息绑定节点。
  - **现状**：`Sidebar.tsx` 目前仅使用 `setTimeout` 模拟回复，完全未对接 LLM，也未获取任何上下文信息。
- **客户端联网检索流程**：
  - **文档要求**：客户端在生成节点前，**必须先进行联网搜索**，并将检索结果作为上下文发送给 LLM。
  - **现状**：`NodeCreateDialog.tsx` 直接调用 `generateNode`，未在前端执行 `apiClient.search` 检索步骤，也未将检索结果注入请求上下文。
- **交互式动画生成**：
  - **文档要求**：选中文本后生成交互式动画网页HTML并内嵌。
  - **现状**：`NodeDetailView.tsx` 中有相关 Case，但仅更新了数据字段，未见实际调用 LLM 生成 HTML 及前端渲染/沙箱执行的逻辑。

### 3. 内容渲染与安全
- **代码执行沙箱**：
  - **文档要求**：支持可运行代码段（`run:python` 等），必须在**后端受限容器**或**沙箱 iframe** 中执行。
  - **现状**：`NodeDetailView.tsx` 仅实现了基础 Markdown 渲染，未见针对特定代码块（`run:xxx`）的解析、执行按钮及沙箱环境实现。
- **Mermaid 与 SVG 编辑**：
  - **文档要求**：自动渲染 Mermaid 图，支持 SVG 直接嵌入并可编辑。
  - **现状**：Markdown 渲染器中未集成 `remark-mermaid` 或类似插件，不支持图表渲染。

### 4. 数据与持久化
- **服务端持久化**：
  - **文档要求**：支持 REST/GraphQL API 存储到数据库（Postgres/MongoDB）。
  - **现状**：目前仅使用 `localforage` (IndexedDB) 进行客户端本地存储，`api.ts` 中虽有接口定义但主要依赖 Mock 或本地服务，未见完整的服务端同步逻辑。
- **导出与导入**：
  - **文档要求**：支持导出为 JSON、Markdown、OPML、GraphML。
  - **现状**：Store 中有 `export` 模态框状态，但未见具体导出功能的实现代码。

### 5. 其他细节
- **联想推荐交互**：
  - **文档要求**：生成的新节点应在画布上显示生成进度。
  - **现状**：`IdeaDialog.tsx` 虽然实现了上下文构建，但在创建节点时也是在后台异步更新状态，未配合前台节点卡片的进度遮罩 UI（同第一点）。

**总结**：项目目前的骨架（画布、节点树结构、基础Markdown渲染）已搭建完毕，但**AI智能流（检索->上下文->生成）**、**高级渲染（代码运行/导图/动画）**以及**服务端交互**仍处于待实现阶段。
